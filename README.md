# Store Analytics Pro

Streamlit-дашборд для анализа экспортов заказов интернет-магазина и управления CRO-бэклогом.

## Что улучшено

- Дни без заказов учитываются в средней дневной выручке и сравнении периодов.
- Повторяющиеся `order_id` и записи без даты не искажают отчёт: они исключаются и явно показываются в блоке качества данных.
- Динамика товара не рисует ложные «+100%», когда в первой половине периода не было продаж: вместо этого показывается отсутствие сопоставимой базы.
- При сбое Google Sheets CRO-раздел не показывает устаревшие fallback-цифры как актуальные.
- Логика импорта, аналитики и интерфейса разделена по модулям; базовые тесты защищают расчёт календарных дней.

## Запуск

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

## Структура

```text
app.py          # интерфейс и визуализации
parser.py       # безопасный разбор XML
analytics.py    # расчёты, не зависящие от Streamlit
cro_module.py   # CRO-чек-лист из Google Sheets
tests/          # проверки ключевых расчётов
```

## Ограничения данных

Проект рассчитан на XML со структурой заказа, близкой к OpenCart-экспорту: `item`, `order_id`, `date_added`, `products/product`. В отчёт попадают только заранее заданные статусы. Сумма заказа может отличаться от суммы товарных строк из-за доставки, скидок или комиссий — это отдельно отражено в «Качестве данных».

Файлы заказов не записываются приложением на диск. Не добавляйте реальные выгрузки с персональными данными в GitHub.
