# Store Analytics MVP

Веб-дашборд для анализа XML-файлов с заказами OpenCart и WooCommerce.

## Что работает

- загрузка XML через браузер;
- безопасный разбор XML;
- учет только трех разрешенных статусов;
- фильтр по датам и статусам;
- сумма заказов;
- количество заказов;
- средний чек;
- количество проданных единиц;
- покупатели и повторные покупатели;
- динамика продаж;
- топ-5 товаров по количеству;
- топ-5 товаров по сумме;
- аналитика статусов, оплаты и регионов;
- таблица товаров;
- выгрузка CSV;
- проверка расхождений данных.

## Разрешенные статусы

- Обробляється менеджером
- Успішна оплата LiqPay
- Очікування

Все другие статусы исключаются до расчета статистики.

## Локальный запуск

### Windows

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

### Linux и macOS

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

После запуска откройте адрес, который покажет Streamlit. Обычно это:

```text
http://localhost:8501
```

## Развертывание в интернете

1. Создайте GitHub-репозиторий.
2. Загрузите файлы проекта.
3. Подключите репозиторий в Streamlit Community Cloud.
4. Укажите `app.py` как основной файл.

## Для полноценного SaaS позже потребуются

- авторизация;
- отдельный кабинет каждого клиента;
- несколько магазинов у одного клиента;
- PostgreSQL;
- история импортов;
- защита от дублей по `store_id + order_id`;
- обработка больших XML через очередь;
- роли пользователей;
- подключение OpenCart и WooCommerce через API;
- отдельный backend.
