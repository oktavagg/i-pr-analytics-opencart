# I-PR Store Analytics MVP

Streamlit-дашборд для анализа заказов и каталога интернет-магазина.

## Обязательный импорт при первом входе

До открытия интерфейса пользователь загружает два XML-файла:

1. XML заказов.
2. XML полного каталога товаров.

Приложение проверяет структуру обоих файлов и только после успешной проверки открывает аналитику и CRO-модуль.

Загруженные файлы сохраняются в каталоге `uploaded_data` и используются при следующих обновлениях страницы. Удаление выполняется отдельной кнопкой в боковой панели. После удаления снова открывается стартовый экран импорта.

## Структура проекта

```text
store-dashboard/
├── app.py
├── conclusions.py
├── orders.py
├── customers.py
├── products.py
├── analytics_ui.py
├── xml_parser.py
├── parser.py
├── cro_module.py
├── requirements.txt
├── README.md
├── .gitignore
├── ipr.jpeg                  # необязательно
└── .streamlit/
    └── config.toml
```

`parser.py` оставлен как совместимый переходный файл. Основной модуль разбора XML называется `xml_parser.py`. Такое имя исключает конфликт со старыми или сторонними модулями `parser`.

## Локальный запуск

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

Linux и macOS:

```bash
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

## Streamlit Community Cloud

Загрузите в репозиторий все файлы из архива. В настройках приложения укажите `app.py` как Main file path.

Важно обновить одновременно `app.py` и `xml_parser.py`. Старый `parser.py` больше не используется приложением напрямую.

## Новая структура аналитики

После импорта файлов аналитика работает как набор отдельных страниц. Навигация находится слева и разделена на группы:

- выводы и рекомендации;
- заказы;
- покупатели;
- товары.

Заголовки групп не нажимаются. Каждый вложенный пункт открывает отдельную страницу с одним показателем, графиком или таблицей.

Подключены существующие расчеты: изменения за период, рекомендации, оборот, количество заказов, средний чек, товары в заказе, статусы, покупатели, повторные продажи, заказы на покупателя, топ товаров и совместные покупки.

Для будущих модулей остались страницы-заглушки в разделах покупателей и товаров. Раздел «Заказы» реализован полностью.

## Обновление v9: сравнение периодов

Страница «Изменения за период» переработана в отдельную сравнительную сводку.

- диапазон выбирается прямо в верхней части страницы;
- предыдущий период такой же длины рассчитывается автоматически;
- выводятся оборот, заказы, средний чек, новые и повторные заказы;
- добавлено распределение заказов по количеству товаров;
- добавлено количество уникальных покупателей;
- для каждого показателя выводятся изменение в процентах и текстовый вывод;
- страница «Рекомендации» оставлена без изменений.


## Обновление v10: модульная структура

Аналитические страницы разделены по категориям бокового меню:

- `conclusions.py`, выводы, сравнение периодов и рекомендации;
- `orders.py`, все страницы раздела заказов;

- `products.py`, все страницы раздела товаров;

`app.py` отвечает за импорт XML, общую подготовку данных, навигацию и запуск нужного модуля.

Также изменен цвет заголовков таблицы сравнения на фирменный желтый и исправлено повторное открытие боковой панели после сворачивания.

## Обновление v11: отчёты по заказам

Четыре страницы раздела «Заказы» переработаны в месячные отчёты:

- «Оборот»: общий оборот, количество заказов, средний чек, таблица и график по месяцам;
- «Оборот от новых/старых»: оборот новых и повторных покупателей, доли, таблица и групповой график;
- «Количество заказов»: сводные показатели, таблица и график по месяцам;
- «Количество заказов от новых/старых»: новые и повторные заказы, доли, таблица и групповой график.

Отчёты используют период и статусы из боковых фильтров. Неполный месяц рассчитывается только по датам выбранного диапазона. Новый покупатель определяется по первому заказу в загруженной истории.

В разделе «Рекомендации» добавлены уровни приоритета:

- «Критично»;
- «Важно»;
- «Рекомендация»;
- «Идея».

## Обновление v13: раздел «Покупатели» завершён

Страница «Количество (новые/старые)» оставлена без изменений.

Добавлены полноценные отчёты для остальных страниц категории:

- «Доля повторных»: оборот и количество новых и повторных заказов, доли и круговая диаграмма;
- «Заказов на покупателя»: среднее количество заказов, доля повторных клиентов, распределение частоты и CRO-действие;
- «Спящие покупатели»: клиенты с двумя и более покупками без заказа 90 дней, контактная таблица и экспорт CSV;
- «ТОП-10 клиентов по обороту»: рейтинг, контакты, количество заказов, оборот, средний чек и график;

Все расчёты используют реальную историю заказов из загруженного XML, выбранный период и отмеченные статусы.

## Обновление v14: раздел «Товары» завершён

Каталог товаров теперь разбирается в отдельную таблицу с остатками, ценой, просмотрами, статусом, производителем и ссылкой на карточку.

Добавлены полноценные страницы:

- «Товары без продаж»: активные товары без продаж в выбранном периоде, CRO-сегменты, остатки, просмотры, последняя продажа и CSV-экспорт;
- «Топ товаров по выручке»: лидер, оборот топ-10, доля в общем обороте, таблица и горизонтальный график;
- «Топ товаров по количеству продаж»: рейтинг по числу заказов с товаром, проданные единицы, оборот, таблица и график;
- «Покупают вместе»: главная пара, доля среди заказов с несколькими товарами, три основные связки, график и таблица.

Страница товаров без продаж разделяет позиции на четыре сегмента:

- высокий интерес, но нет продаж;
- никогда не продавался;
- нет продаж в выбранном периоде;
- нет остатка.

Все расчёты используют загруженный XML каталога, XML заказов, выбранный период и отмеченные статусы.
