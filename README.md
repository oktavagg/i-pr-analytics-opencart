# I-PR Store Analytics MVP

Белый веб-дашборд для анализа XML-файлов с заказами OpenCart и WooCommerce.

## Логотип

Добавьте файл `ipr.jpeg` в корень проекта рядом с `app.py`.

Структура репозитория:

```text
store-dashboard/
├── app.py
├── parser.py
├── requirements.txt
├── ipr.jpeg
├── .gitignore
└── .streamlit/
    └── config.toml
```

Если `ipr.jpeg` отсутствует, приложение продолжит работать и покажет подсказку вместо логотипа.

## Что работает

- загрузка XML через браузер;
- безопасный разбор XML;
- учет только статусов:
  - `Обробляється менеджером`;
  - `Успішна оплата LiqPay`;
  - `Очікування`;
- фильтры по датам и статусам;
- сумма заказов, количество заказов, средний и медианный чек;
- проданные единицы и среднее количество товаров в заказе;
- уникальные и повторные покупатели;
- доля суммы от повторных клиентов;
- динамика продаж и сравнение половин периода;
- топ товаров по количеству и сумме;
- рост и снижение продаж отдельных товаров;
- товары, которые часто покупают вместе;
- анализ статусов, оплаты, регионов и дней недели;
- автоматические бизнес-рекомендации;
- отчет по товарам в CSV;
- проверка качества данных.

## Локальный запуск

```bash
python -m venv .venv
```

Windows:

```bash
.venv\Scripts\activate
pip install -r requirements.txt
streamlit run app.py
```

Linux и macOS:

```bash
source .venv/bin/activate
pip install -r requirements.txt
streamlit run app.py
```

## Развертывание через Streamlit Community Cloud

1. Распакуйте архив.
2. Добавьте `ipr.jpeg`.
3. Загрузите файлы в корень GitHub-репозитория.
4. Подключите репозиторий в Streamlit Community Cloud.
5. Укажите `app.py` как Main file path.

XML-файлы не нужно загружать в GitHub.

## CRO-модуль

Добавлен отдельный файл `cro_module.py`.

В левом меню доступны два раздела:

- `Аналитика`, текущий дашборд заказов;
- `CRO`, сводка по CRO-чек-листу и встроенная Google-таблица.

CRO-модуль работает независимо от загрузки XML. Google Sheets остается источником данных. Сводка обновляется каждые 5 минут или вручную кнопкой в интерфейсе.

## Стартовый экран загрузки

В разделе аналитики дашборд скрыт до загрузки XML. Пользователь сначала видит отдельный анимированный экран загрузки. После выбора файла приложение автоматически открывает аналитику.

Переключение между `Аналитикой продаж` и `CRO-аудитом` осталось слева, но оформлено как две крупные контрастные кнопки.
